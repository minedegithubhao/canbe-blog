# 大模型应用开发体系

## 大模型的基本概念

### **一、大模型是什么？**

> **大模型：**是一个基于海量数据训练、拥有巨大参数量的深度学习神经网络，核心能力是理解和生成人类语言，并能基于此完成各类复杂任务，**简单来说**就是一个 读过海量文本（书籍、网页、文章、代码等）的超级大脑。

### **二、为什么取名为大模型？**

**“大”：**主要体现如下几个方面：

**参数量大**

- 早期模型：百万、千万级参数
- 现在大模型：**百亿、千亿、甚至万亿级参数**
- 参数越多，模型容量越大，越能记住知识、捕捉复杂语言规律

**训练数据量大**

- 训练语料通常是**几十亿到几万亿 tokens（词 / 字单位）**
- 包含书籍、网页、论文、代码、对话等，覆盖广泛知识

**模型结构大、计算量大**

- 基于 **Transformer 架构**（2017 年谷歌提出）
- 训练需要大量 GPU/TPU，成本极高

**“模型”：**它的本质是概率模型：根据上文，预测下一个最可能出现的词 / 字，从而生成整段内容

### 三、基本应用场景？

- 文本处理
- **对话系统**
- **检索增强生成（RAG）：**利用外部知识库，获取上下文信息，帮助模型更好的理解用户需求，生成更准确的内容
- **多模态内容处理**
  - 图像处理
  - 语音和视频处理

### 四、大模型能做什么？

- 智能助手：ChatGPT、豆包、文心一言等
- 办公提效：写文档、做表格、生成 PPT 大纲、总结会议
- 开发辅助：写代码、查 Bug、生成接口文档
- 教育：答疑、辅导、出题、批改
- 内容创作：文案、小说、剧本、短视频脚本
- 企业应用：客服、知识库问答、数据分析、流程自动化

### 五、大模型的工作原理

1. **预训练（Pre-training）**
   - 用海量文本 “自学” 语言规律、知识、逻辑
   - 目标：预测下一个词，让生成的句子通顺合理
2. **微调（Fine-tuning）**
   - 用高质量对话 / 指令数据再训练
   - 让模型更听话、更符合人类意图
3. **对齐（Alignment）**
   - 让模型输出更安全、有用、无害
   - 避免胡说、偏见、有害内容

### 六、大模型的特点与局限

**优点**

- 泛化能力强：一个模型能做 N 种任务，不用每个任务单独训练
- 零样本 / 少样本学习：给个例子就能做，不用大量标注数据
- 交互自然：用自然语言就能指挥它做事

**局限**

- **幻觉（Hallucination）**：会一本正经地胡说八道
- 知识有时效性：训练数据截止前的知识，之后的不知道
- 逻辑深度有限：复杂推理、数学、专业领域仍会出错
- 无法真正 “理解” 与 “意识”：只是统计拟合

### 七、常见的大模型

**开源 / 可本地部署**：Llama 3、Mistral、Qwen（通义千问开源）、GLM、Baichuan 等

**闭源 / 在线服务**：GPT-4o、Claude 3、Gemini、文心一言、通义千问、豆包等
